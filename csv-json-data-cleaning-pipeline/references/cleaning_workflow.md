# 清洗流程说明

## 1. 文档目的

本文档用于说明 CSV/JSON数据批量清洗产线的标准处理流程，帮助数据开发人员按照统一步骤完成数据加载、规则校验、字段检查、去重、缺失处理、异常检测、标准化、问题导出和统计报告生成。

该流程适用于 CSV、JSON、JSONL 等结构化和半结构化数据，不适用于图像、音频、视频等多模态数据清洗。

## 2. 总体流程

推荐清洗流程如下：

1. 数据加载。
2. 规则校验。
3. 字段检查。
4. 去重处理。
5. 缺失值处理。
6. 异常值处理。
7. 格式标准化。
8. 问题数据导出。
9. 统计报告生成。

各步骤应记录处理结果，关键步骤应写入 `cleaning_log`，问题记录应写入 `issue_rows`。

## 3. 详细流程

### 3.1 数据加载

目标：读取原始数据文件，识别文件类型、字段结构和记录数量。

输入：

- CSV 文件。
- JSON 文件。
- JSONL 文件。
- 文件编码和分隔符配置。

处理要求：

- CSV 默认第一行为表头。
- JSON 推荐为对象数组。
- JSONL 要求每行一个 JSON 对象。
- 文件读取失败时，应停止清洗并返回错误原因。

输出：

- 原始数据记录。
- 字段列表。
- 输入记录总数。
- 文件读取日志。

### 3.2 规则校验

目标：检查清洗规则配置是否完整、合理、可执行。

输入：

- `rule_template.yaml` 或用户自定义规则文件。
- 原始数据字段列表。
- 可选字段映射文件。
- 可选数据字典文件。

处理要求：

- 检查必需配置项是否存在。
- 检查规则中引用的字段是否存在。
- 检查处理动作是否合法。
- 检查枚举值、日期格式、数值范围等配置是否完整。

输出：

- 规则校验结果。
- 无效规则列表。
- 规则校验日志。

### 3.3 字段检查

目标：检查数据字段是否符合规则要求。

处理内容：

- 检查必填字段是否存在。
- 检查字段名是否需要映射。
- 检查每条记录是否缺少关键字段。
- 检查 JSON 或 JSONL 记录字段是否结构不一致。

处理建议：

- 缺少必填字段时，应写入问题数据。
- 字段名可映射时，按映射表转换。
- 不影响使用的多余字段可保留或按配置删除。

输出：

- 字段检查结果。
- 字段缺失问题记录。
- 字段映射结果。

### 3.4 去重处理

目标：识别重复数据并按规则处理。

处理内容：

- 按单字段去重，如 `id`。
- 按组合字段去重，如 `title + source`。
- 记录重复组和重复数量。

处理策略：

- `first`：保留第一条。
- `last`：保留最后一条。
- `latest`：保留更新时间最新的一条。
- `export_only`：仅导出重复问题，不删除。

输出：

- 去重后的数据。
- 重复数据问题清单。
- 去重统计。

### 3.5 缺失值处理

目标：识别并处理空值、默认占位值和无效缺失值。

常见缺失值：

- 空字符串。
- `null`。
- `None`。
- `N/A`。
- `未知`。
- `无`。

处理策略：

- `drop`：删除记录。
- `fill`：填充默认值。
- `mark`：标记问题。
- `ignore`：忽略。

处理建议：

- 关键字段缺失优先标记或删除。
- 可补充字段可使用默认值填充。
- 所有填充和删除动作应写入日志。

输出：

- 缺失值处理后的数据。
- 缺失值问题记录。
- 缺失值处理统计。

### 3.6 异常值处理

目标：根据规则识别不符合业务要求的数据。

常见异常：

- 数值超出范围。
- 日期无法解析。
- 日期逻辑错误。
- 字段长度过短或过长。
- 枚举值不在允许范围内。
- 内容为空、过短或明显无意义。

处理策略：

- 可修复的异常执行标准化。
- 不可修复的异常写入 `issue_rows`。
- 严重异常可按配置删除。

输出：

- 异常检测结果。
- 异常问题记录。
- 异常统计。

### 3.7 格式标准化

目标：统一不同来源数据的字段格式。

常见标准化内容：

- 日期格式标准化。
- 手机号格式标准化。
- 金额格式标准化。
- 枚举值标准化。
- 空格、换行、特殊字符处理。

处理建议：

- 标准化前应保留原始值。
- 标准化失败时应写入问题数据。
- 标准化规则应写入日志。

输出：

- 标准化后的数据。
- 标准化失败记录。
- 标准化统计。

### 3.8 问题数据导出

目标：将清洗过程中发现的问题数据统一导出，便于人工复核。

建议输出字段：

- 原始行号或记录编号。
- 问题类型。
- 问题字段。
- 原始值。
- 问题原因。
- 处理动作。
- 规则名称。

处理要求：

- 同一条记录可存在多个问题。
- 问题记录应尽量保留原始上下文。
- 导出文件应便于人工查看和二次处理。

### 3.9 统计报告生成

目标：生成清洗结果统计和过程日志。

`cleaning_summary` 建议包含：

- 输入记录数。
- 输出记录数。
- 问题记录数。
- 删除记录数。
- 修复记录数。
- 各类问题数量。
- 各规则执行结果。

`cleaning_log` 建议包含：

- 执行步骤。
- 规则名称。
- 执行时间。
- 影响记录数。
- 执行结果。
- 说明信息。

## 4. 异常处理原则

1. 文件无法读取时，停止任务并输出错误原因。
2. 规则配置错误时，优先提示配置问题，不继续执行清洗。
3. 字段不存在时，应说明缺失字段名称。
4. 自动修复有风险时，应标记问题，不应强行修改。
5. 删除数据必须记录数量和原因。
6. 所有输出文件应保持命名清晰、结构稳定。

## 5. 最小可用流程

如果先做最小可用版本，建议只实现以下流程：

1. CSV 和 JSONL 数据加载。
2. 规则文件读取和基本校验。
3. 必填字段检查。
4. 主键去重。
5. 缺失值标记。
6. 问题数据导出。
7. 清洗后数据输出。
8. 清洗统计生成。

## 6. 验收关注点

| 验收项 | 关注点 |
|---|---|
| 数据加载 | 文件能读取，字段和记录数识别正确。 |
| 规则校验 | 错误配置能被发现并说明原因。 |
| 字段检查 | 缺失字段和空值能被正确识别。 |
| 去重处理 | 重复记录能被识别并按规则处理。 |
| 问题导出 | 问题数据字段完整，原因清晰。 |
| 清洗统计 | 输入、输出、问题、删除、修复数量准确。 |
| 日志追溯 | 每个关键步骤都有执行记录。 |
